# Phase 1: 选题与诊断

## 目标
判断一个点子适不适合写成中文网文，以及适合什么体量。

## 执行流程

### 1. 题材诊断六问
收到点子后，先诊断这六项：

1. **题材是什么？** （都市/仙侠/奇幻/言情/悬疑/历史/科幻/系统）
2. **核心卖点是什么？** （读者为什么点进来？）
3. **核心冲突是什么？** （主角最紧迫的麻烦？）
4. **读者来这本书主要想吃什么？** （爽/甜/虐/悬疑/热血）
5. **故事引擎是什么？** （升级/解密/复仇/经营/关系拉扯）
6. **更适合什么体量？** （短篇1-3w/中篇10-20w/长篇50w+/连载100w+）

### 2. 好坏判断

**好点子的特征（越多越好）：**
- 一句话能说清卖点
- 冲突具体，不是空泛成长
- 主角处境有压迫感
- 设定会直接制造事件
- 不靠解释也能立住看点

**坏点子的特征（警惕）：**
- 设定很大，事件很弱
- 题材很混，消费点不明
- 主角只是"很特别"，但没有被逼着行动
- 全靠后期反转撑，不会跑前10章

### 3. 输出格式

```markdown
## 选材诊断书

题材：{仙侠/都市/言情/...}
卖点：{一句话卖点}
冲突：{核心冲突}
读者期待：{读者情感需求}
故事引擎：{驱动类型}
推荐体量：{短篇/中篇/长篇/连载}

判断结论：✅ 值得写 / ⚠️ 需要调整 / ❌ 不建议
```

### 4. 可选：扫榜参考
如用户有明确平台目标（起点/番茄/晋江），参考以下数据：
- 该平台当前热门题材
- 类似作品的切入点
- 市场容量判断
