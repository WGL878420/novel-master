# Phase 6: 校验与修复

## 目标
自动检查所有章节完成度，不合格章节自动重写（最多3轮）。

## 原则
全程无需用户介入，自动完成。

## 执行流程

### 1. 完整性检查
- 检查所有计划章节是否已完成
- 检查每章文件是否存在、不为空

### 2. 批量门禁检查
```bash
python3 scripts/quality_gate.py report
```

逐章检查五项指标，汇总不合格章节。

### 3. 节奏健康度检查
```bash
python3 scripts/event_matrix.py rhythm
```

检查事件分布是否合理，是否存在连续同类型事件或长时间无情感/世界观事件。

### 4. 知识图谱一致性检查
```bash
python3 scripts/story_graph.py check
```

### 5. 伏笔追踪
检查所有标记为"open"的伏笔：
- 是否在预期章节内被回收
- 未被回收的伏笔是否有合理规划

### 6. 自动修复流程

对不合格章节执行（最多3轮）：

```
第1轮：修复门禁失败项（字数/钩子/冲突/AI味）
第2轮：重新检查，如仍不合格则再次修复
第3轮：最终修复，再不通过则标记为"需人工介入"
```

### 7. 最终报告

```markdown
## 最终校验报告

总章节：{X}
通过：{Y}
条件通过（有警告）：{Z}
失败（需人工修复）：{W}

### 各章详情
- 第001章: ✅ 全部通过
- 第002章: ⚠️ AI味偏高(4.2/千字)
- 第003章: ❌ 字数不足(1500)

### 节奏评估
✅ 事件分布合理
✅ 无长时间缺失情感/世界观章节
```
