# Phase 3: 结构规划

## 目标
将创意转化为可执行的结构化大纲，确保故事有骨架、有走向、有节点。

## 执行流程

### 1. 定义故事引擎
明确故事的驱动力：
- 升级流：主角不断变强，遇强则强
- 解密流：逐步揭开真相
- 关系流：人物关系的变化与拉扯
- 经营流：建设/经营/扩张
- 复仇流：逐步清算
- 求生流：在危机中存活
- 混合引擎：主引擎 + 副引擎

### 2. 选择叙事结构
参考 [guides/story-structures.md](../guides/story-structures.md) 选择合适结构：
- 三幕式（通用，推荐）
- 英雄之旅（冒险/奇幻）
- 悬疑结构（侦探/推理）
- 言情结构（爱情题材）
- 多线叙事（复杂剧情）

### 3. 分卷规划
按体量拆分：

| 体量 | 卷数 | 每卷章数 |
|------|------|---------|
| 短篇(10章) | 1卷 | 10章 |
| 中篇(30章) | 3卷 | 每卷10章 |
| 长篇(60章) | 4-6卷 | 每卷10-15章 |

每卷定义：
- 本卷核心目标
- 本卷最强冲突
- 本卷最大的兑现点
- 卷末局面变化

### 4. 章纲编写
每章最低配置：
- **本章目标**：主角这章想做什么
- **本章冲突**：谁或什么阻止他
- **本章新信息**：读者这章知道了什么
- **章末拉力**：为什么读者还要往下看

使用 [outline-template.md](../guides/outline-template.md) 生成七栏大纲。

### 5. 人物档案
为每个主要角色建立档案（使用 [character-building.md](../guides/character-building.md) 的模板）：
- 基本信息（姓名/年龄/身份）
- 核心动机（他要什么？为什么？）
- 核心缺陷（阻止他得到目标的内在障碍）
- 人物弧线（故事开始→结束的变化）
- 与其他角色的关系

### 6. 写作计划JSON
生成机器可读的写作计划：

```json
{
  "title": "小说名称",
  "total_chapters": 30,
  "total_volumes": 3,
  "structure": "three-act",
  "engine": "升级+解密",
  "volumes": [
    {
      "number": 1,
      "title": "第一卷名称",
      "chapters": "01-10",
      "goal": "本卷目标",
      "climax": "卷末高潮"
    }
  ],
  "characters": ["角色ID列表"],
  "current_chapter": 0
}
```

## 输出
- `bible/03-outline.md` — 完整大纲（含分卷+章纲）
- `bible/01-character-profiles.md` — 人物档案
- `02-写作计划.json` — 机器可读写作计划

等待用户确认后进入 Phase 4。
