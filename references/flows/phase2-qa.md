# Phase 2: 三层问答

## 目标
通过递进式问答收集创作需求，确定小说核心定位与标题。

## 原则
- 不要让用户填问卷地狱
- 信息够用就直接跳过，只补最少量澄清
- 每层都支持「跳过/默认」

## 执行流程

### Layer 1: 核心定位（必答，3问）

**Q1: 题材与创意**
- 如果 Phase 1 已确定题材，此处直接确认
- 否则问：想写什么题材的小说？（仙侠/都市/言情/悬疑/历史/科幻/奇幻/其他）
- 支持 🎲 随机推荐

**Q2: 主角设定**
- 主角是什么身份？（穿越者/重生者/普通人/天才/废柴/...）
- 主角的核心特质是什么？（性格/能力/缺陷）

**Q3: 核心冲突**
- 故事开局时主角面临的最大麻烦是什么？
- 这个麻烦会持续驱动多久？

### Layer 2: 深度定制（可选，5问）

**Q4: 世界观**
- 世界背景/时代/阵营设定
- 如不需要则跳过

**Q5: 叙事视角与基调**
- 第一人称/第三人称
- 轻松/正剧/压抑/热血

**Q6: 核心主题**
- 这个故事最终想表达什么？
- 如不需要则跳过

**Q7: 目标读者**
- 男频/女频/全民向
- 预期阅读场景（通勤/睡前/追更）

**Q8: 章节数量**
- 推荐值：短篇10章 / 中篇30章 / 长篇60章 / 连载不限
- 支持跳过（用默认值）

### Layer 3: 标题生成

基于以上信息，AI 生成 3-5 个候选标题，用户选择或自定义。

标题要求：
- 包含卖点关键词
- 3-10 字为宜
- 有辨识度

## 输出
- 确认卡片（所有选择的汇总）
- 选定标题
- 进入 Phase 3
