# 大纲

## 故事引擎
大唐神探系统的核心驱动力是「穿越者用现代刑侦思维破解古代悬案，同时揭开自身穿越真相」。
每3-5章构成一个案件单元，案件之间由暗线串联，逐步指向最终幕后黑手。

### 核心悬念
- 主线：谁在操控长安城的系列案件？目的是什么？
- 暗线1：主角穿越的原因是什么？系统是谁制造的？
- 暗线2：原主身份是否隐藏着秘密？

### 节奏规划
- 每章必须包含：至少1个新线索 + 1次冲突升级 + 1个章末钩子
- 每5章必须包含：1次事件高潮 + 1次世界观/情感补充

## 叙事结构
三幕结构（章节数不设上限，可随剧情自然扩展）

| 幕 | 功能 |
|---|------|
| 第一幕：开局 | 引入世界观、主角困境、初始冲突 |
| 第二幕：发展 | 冲突升级、矛盾深化 |
| 第三幕：终局 | 推向高潮、解决最终冲突 |
