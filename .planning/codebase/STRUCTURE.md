## Directory Purposes

**scripts/:**
- Purpose: All executable CLI tools
- Contains: story_graph.py, quality_gate.py, init_project.py, fanqie_publish.py, anti_ai_detector.py, event_matrix.py, utils.py
- Key files: `scripts/story_graph.py` (knowledge graph + workflow), `scripts/quality_gate.py` (validation), `scripts/utils.py` (shared)

**references/guides/:**
- Purpose: Writing methodology and platform-specific guides
- Contains: chapter-craft.md, anti-ai-flavor.md, anti-ai-flavor-rules.md, fanqie-platform-writing.md, short-story-writing.md, story-structures.md, dialogue-writing.md, character-building.md, etc.
- Key files: `references/guides/fanqie-platform-writing.md` (131篇番茄教程汇总)

**references/flows/:**
- Purpose: Workflow phase documentation
- Contains: phase0-init.md through phase6-validation.md, shared-infrastructure.md

**templates/:**
- Purpose: Project template structure for new novels
- Contains: templates/bible/, templates/manuscript/, templates/plan/, templates/state/
- Note: templates/state/ has same structure as project state/

**corpus/:**
- Purpose: Reference articles for style analysis
- Contains: corpus/articles/, corpus/analysis/

**demo/:**
- Purpose: Example projects demonstrating structure
- Contains: Example novel projects with complete bible, manuscript, state, knowledge/

**.planning/codebase/:**
- Purpose: Architecture documentation for GSD workflow
- Contains: ARCHITECTURE.md, STRUCTURE.md (this file location)

## Key File Locations

**Entry Points:**
- `scripts/init_project.py`: Create new project (`python3 scripts/init_project.py "书名" --genre x`)
- `scripts/story_graph.py`: Knowledge graph and workflow management (subcommands)
- `scripts/quality_gate.py`: Quality validation (`python3 scripts/quality_gate.py check -p <proj>`)
- `scripts/fanqie_publish.py`: Fanqie novel publishing (subcommands)
- `scripts/anti_ai_detector.py`: AI flavor detection
- `scripts/event_matrix.py`: Event rhythm tracking

**Configuration:**
- `scripts/utils.py`: WRITING_TYPES configuration, shared utilities
- No central config file - settings stored in project state.json

**Core Logic:**
- `scripts/utils.py`: File I/O, chapter parsing, Chinese char counting, writing type config
- `scripts/story_graph.py`: Graph operations, brief generation, post-write sync
- `scripts/quality_gate.py`: Five-step validation logic

**Testing:**
- No test files detected - validation is manual or via quality_gate

Each novel project follows this structure:

```

├── bible/                    # Planning documents (generated by init_project.py)

│   ├── 01-character-profiles.md # Character definitions

│   ├── 03-outline.md         # Chapter outline (table format)
│   └── 04-research.md        # Reference research
├── manuscript/               # Written chapters
│   ├── 第001章-xxx.md        # Chapter files
│   └── ...
├── knowledge/                # Graph and timeline data
│   ├── story_graph.json      # Nodes and edges graph
│   ├── event_matrix.json     # Event tracking with cooldowns
│   └── timeline.md           # Chronological events
├── state/                    # Project state
│   ├── current/state.json    # Current project state

    ├── 第001章-gate.json
    └── ...
```

## Naming Conventions

**Files:**
- Scripts: lowercase_with_underscores.py
- Project directories: `{timestamp}-{safe_name}` format (e.g., `20260428-003000-离婚后他后悔了`)
- Chapter files: `第{XXX}章-{title}.md` (zero-padded 3-digit chapter number)
- Bible files: `NN-{name}.md` (2-digit prefix for ordering)
- Gate records: `第{XXX}章-gate.json`
- JSON data files: snake_case.json

**Functions:**
- snake_case (e.g., `check_word_count`, `add_node`, `generate_brief`)
- Private helpers: `_leading_underscore`

**Command-line Arguments:**
- kebab-case for flags (e.g., `--project`, `--chapter`, `--genre`)

**Directories:**
- lowercase (bible/, manuscript/, knowledge/, state/, gates/)
- No spaces in paths

## Where to Add New Code

**New CLI Script:**
- Place in `scripts/` following existing pattern (argparse with subcommands)
- Import from `scripts/utils.py` for shared utilities
- Update CLAUDE.md if new command affects workflow

**New Bible/Planning Document:**
- Create in `bible/` with 2-digit prefix (e.g., `05-lore.md`)
- Update `init_project.py` generate_bible_content() if new bible file should be auto-generated

**New Knowledge Graph Node Type:**
- Add to NODE_TYPES dict in `scripts/story_graph.py`
- Update brief generation to include new type if needed

**New Quality Gate Check:**
- Add function in `scripts/quality_gate.py` following check_xxx pattern
- Add to checks dict in run_full_gate()
- Update print_gate_result() for output formatting

**New Writing Type:**
- Add to WRITING_TYPES dict in `scripts/utils.py`
- Include word_limits or phases configuration

- Committed: Yes
- Note: Graph persisted as JSON, timeline as Markdown

- Purpose: Project state persistence
- Generated: Yes (by init_project.py and various scripts)

- Note: template/state.json is copy of initial state

- Committed: Yes

- Generated: No
- Committed: Yes
- Note: Mirrors project structure, used by init_project.py

- Purpose: Example projects
