### Task 2: 实现 login 子命令（手机验证码）

<read_first>
- `scripts/qidian_publish.py` (Task 1 创建的框架)
- `scripts/fanqie_publish.py` 中 `_check_logged_in`, `_navigate_and_check_login`, `cmd_login` 的实现
</read_first>

<action>
实现 `cmd_login()` 异步函数：

1. 启动浏览器，加载已有认证状态（如有）
2. 导航到 `https://write.qq.com` 作家后台
3. 检测是否已登录（复用番茄的 `_check_logged_in` 模式，但需适配起点页面文字）
4. 如未登录，等待用户输入手机号并完成验证码输入（通过页面元素交互）
5. 登录成功后保存认证状态到 `~/.novel-master/qidian-auth-state.json`

**起点登录页面关键元素（需探索确认）：**
- 手机号输入框
- 验证码发送按钮
- 验证码输入框
- 登录提交按钮

**提示：** 使用 Playwright 的 `page.locator()` 定位元素，`page.fill()` 填值，`page.click()` 点击。验证码需要用户查看手机后手动输入，脚本应提供 input() 提示让用户输入验证码。
</action>

<acceptance_criteria>
- [ ] `python3 scripts/qidian_publish.py login` 可以启动浏览器并导航到起点作家后台
- [ ] 已登录状态可以被正确识别
- [ ] 未登录时脚本会提示输入手机号和验证码
- [ ] 登录成功后 auth state 被保存
</acceptance_criteria>

### Task 3: 实现 upload 子命令（指定章节上传）

<read_first>
- `scripts/qidian_publish.py` (Task 1-2 完成的框架)
- `scripts/fanqie_publish.py` 中 `_parse_chapter_browser`, `_upload_via_browser`, `cmd_upload` 的实现
- `scripts/utils.py` 中 `chapter_number`, `count_chinese_chars`, `get_chapter_files` 等工具函数
</read_first>

<action>
实现 `cmd_upload()` 异步函数：

1. 加载项目状态和认证状态
2. 解析指定 `--chapter N` 的章节文件（用 `get_chapter_files` + `chapter_number` 过滤）
3. 调用 `_parse_chapter_browser()` 解析 Markdown 转 HTML（复用番茄的转换逻辑）
4. 通过浏览器导航到起点新建章节草稿页：
   `https://write.qq.com/portal/booknovels/chaptertmp/CBID/35730802204107809/addType/1.html`
5. 在页面表单中填写章节标题和正文（探索起点页面结构）
6. 点击保存按钮
7. 记录上传结果到 `state/qidian-publish-state.json`

**起点页面探索重点：**
- 章号输入框
- 标题输入框
- 正文编辑器（可能是 textarea 或 contenteditable）
- 保存/存草稿按钮
- 上传成功后是否有提示

**注意：** 起点页面结构未知，需要根据实际页面调整选择器。
</action>

<acceptance_criteria>
- [ ] `python3 scripts/qidian_publish.py upload -p <项目> --chapter N` 可以上传指定章节
- [ ] 上传后状态被记录到 `state/qidian-publish-state.json`
- [ ] 已上传章节不会重复上传（通过状态判断）
</acceptance_criteria>
